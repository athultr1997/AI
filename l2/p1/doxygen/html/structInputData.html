Detailed Description

Data structure to store the input data for the coal allocation problem.

It stores the data as a 3D array.

Member Data Documentation

◆ bid_data

int*** InputData::bid_data

3D array to store the input data. The first dimension corresponds to the company id (cid). The second dimension corresponds to the bids of the particular company set by the first dimension. The first instance in the third dimension is the bid value and the rest corresponds to the block id of the coal. Example usages:
bid_data[i]: all the bid data corresponding to the company defined by id i+1
bid_data[i][j]: details of (i+1)-th company's j-th bid
bid_data[i][j][0]: bid value
bid_data[i][j][k]: block id of the coal (where k>0)
